========================================================================
SRMS — Student Result Management System
Technology Stack and Architecture Specifications
========================================================================

1. OVERVIEW
-----------
SRMS is a lightweight, high-performance, responsive student result management
system. It features independent Admin/Staff and Student dashboards. The
application is designed with a premium, dark-mode-first aesthetic and features
dynamic interactive grade visualization.

2. CORE LANGUAGES
-----------------
* HTML5: Semantic markup, dynamic DOM structures.
* CSS3: Custom CSS variables/tokens for the theme, CSS flexbox/grid for layouts,
  smooth hover micro-animations, glassmorphism card designs, and dynamic toast
  notifications.
* JavaScript (ES6+): Client-side routing, API fetching, authentication flow, 
  and dynamic SVG rendering.
* SQL: SQLite schema definition, data relationships, and query execution.

3. FRONTEND STACK
-----------------
* Style System: Vanilla CSS3 (highly custom variables and modern layout rules).
* Fonts: Inter (imported from Google Fonts).
* Interactive Components:
  - Custom SVG-based GPA Trend Graph (calculated and rendered natively using 
    vanilla JS to maximize performance and minimize dependencies).
  - Custom toast notification container.
  - Interactive class report generator.
* Authentication Handlers: Session token (JWT) management stored locally via
  localStorage.

4. BACKEND STACK
----------------
* Platform: Node.js (v22.18.0+)
* Framework: Express.js (v4.18.3)
* Security & Authentication:
  - JSON Web Tokens (jsonwebtoken v9.0.2) for secure role-based session access.
  - bcryptjs (v2.4.3) for secure password hashing.
  - Middleware-based route protection (Admin only, Staff or Admin, and Student 
    isolation).
* PDF Generation: pdfkit (v0.15.0) used to programmatically generate academic 
  transcripts and semester marksheets, complete with vector seals and tabular grade lists.

5. DATABASE SYSTEM
------------------
* Database: SQLite (srms.db)
* Node Driver: better-sqlite3 (v9.4.3) — a synchronous, high-speed SQLite compiler.
* Schema & Seeding:
  - database.js: Contains schema definitions for:
    - users (admin, staff, student credentials)
    - students (roll numbers, departments, classes, batches)
    - staff (departments, departments assigned)
    - subjects (course code, credits, max marks)
    - results (marks, grades, GPA calculations)
  - seed.js: Fully seeds the database with initial accounts, staff assignments, 
    sample subjects, and mock student grades.

6. UTILITIES & DEVELOPMENT TOOLS
--------------------------------
* nodemon (v3.1.0): Watchdog tool for automatically restarting the Express 
  app upon code changes.
* npm: Project package manager.
